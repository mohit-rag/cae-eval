The handle_reply function in email_handler.py has inline logic for determining the From header when replying from an alias. It resolves which display name to use based on a priority: the alias's own name takes precedence; if the alias has no name but belongs to a custom domain, the domain's name is used instead; if neither is available, the plain email address is used as-is. When a display name is applied, the address is formatted with the name alongside the email in standard email address format. The function also logs which name source was chosen before setting the header.

This name resolution logic is not reusable or testable in isolation. Extract it into a dedicated utility in app/alias_utils.py that accepts an alias and returns the resolved formatted address along with an optional note describing which name source was applied. When no display name is used, there should be no note to report.
